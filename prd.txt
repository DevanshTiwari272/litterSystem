
PRODUCT REQUIREMENTS DOCUMENT
APLMS Monitoring Dashboard
AI-Powered Public Litter Monitoring System


1. Purpose & Scope
This document defines the product requirements for the APLMS Monitoring Dashboard — the primary web-based interface through which city operators, enforcement officers, and administrators interact with the AI-Powered Public Litter Monitoring System.

The dashboard is the operational nerve center of APLMS. It surfaces real-time alerts, incident evidence, analytics, and system health in a single, unified interface — enabling faster response times, smarter resource allocation, and data-driven decision making for urban waste management.



2. User Roles & Permissions
The dashboard supports three distinct roles. Each role sees a tailored interface with only the tools relevant to their responsibilities.



3. Information Architecture
The dashboard is organized into six top-level sections accessible via a persistent left sidebar navigation.



4. Page-by-Page Specifications
4.1 Overview Dashboard (Home)
The Overview page is the default landing page after login. It gives operators an at-a-glance understanding of the system's current state across all zones.

4.1.1 KPI Summary Cards
A row of 4–5 metric cards at the top of the page. Each card shows a current value, a delta versus the previous period (24h or 7d), and a trend sparkline.


4.1.2 Real-Time Alert Ticker
A horizontally scrolling or stacked feed of the 10 most recent unresolved alerts
Each entry shows: camera name, zone, time elapsed since detection, thumbnail, and quick-action button (Acknowledge / Dispatch)
New alerts animate in from the top; resolved alerts fade out
Clicking an alert entry navigates to the full incident detail page

4.1.3 Live Hotspot Map
Interactive map (Leaflet.js / Mapbox GL) showing all camera locations as markers
Marker color encodes alert status: Green = No alerts, Amber = 1–3 alerts, Red = 4+ alerts
Heatmap overlay (toggleable) shows incident density for the selected time range
Clicking a camera marker opens a popover with: camera name, zone, stream thumbnail, incident count today, and link to Live Monitoring view
Time range filter: Last 1h / 6h / 24h / 7d / Custom

4.1.4 Recent Incidents Mini-Table
Last 10 incidents in a compact table: Time, Camera, Zone, Object Detected, Action Detected, Confidence, Status
'View All' button links to the full Incidents page

4.2 Live Monitoring Page
Displays live video feeds from all active cameras with AI detection overlays rendered directly on the streams.

4.2.1 Camera Grid Layout
Default layout: 2×2 grid (4 cameras). Toggle to 3×3 (9 cameras) or 1×1 (single fullscreen)
Each cell shows: camera name, zone label, live stream, bounding box overlays when objects are detected, and a live confidence badge
Cells with active alerts pulse with a red border
User can drag-rearrange cells and save a preferred layout per session

4.2.2 Camera Feed Cell Controls
Fullscreen toggle — expand any camera to fill the viewport
Snapshot button — capture and save the current frame as an incident image
Mute/unmute audio (if camera supports audio stream)
Stream quality selector: Auto / 480p / 720p / 1080p
Detection overlay toggle: show/hide bounding boxes and labels

4.2.3 Camera Selector Sidebar
List of all configured cameras grouped by zone
Drag camera from the list into the grid to swap/add it
Status indicator per camera: Online (green), Offline (red), Reconnecting (amber)

4.3 Incidents Page
A comprehensive log of every detected littering event. This is the primary evidence and audit trail view used by operators and enforcement officers.

4.3.1 Incident List Table

4.3.2 Filters & Search
Global search bar: search by Incident ID, camera name, zone, object type
Date range picker: Today / Yesterday / Last 7d / Last 30d / Custom range
Filter by Zone (multi-select dropdown)
Filter by Camera (multi-select dropdown, dependent on zone filter)
Filter by Status: All / Open / Acknowledged / Resolved / False Positive
Filter by Confidence: All / High (≥80%) / Medium (60–79%) / Low (<60%)
Filter by Object Type: All / Plastic Bottle / Wrapper / Bag / Can / Other

4.3.3 Bulk Actions
Multi-select rows via checkboxes
Bulk Acknowledge, Bulk Assign to Officer, Bulk Mark as False Positive
Export selected rows as CSV or PDF report

4.3.4 Incident Detail Page  (/incidents/:id)
Clicking any incident row opens a dedicated detail page with full context:
Large incident frame image with annotated bounding boxes and labels
Metadata panel: Incident ID, timestamp, camera, GPS coordinates, confidence score, model version
Short video clip (5–10 seconds around detection time) if recording is available
Detection breakdown: list of all detected objects with individual confidence scores
Status change timeline: audit log of who changed the status and when
Assign to officer dropdown + optional notes field
Evidence download button: package frame + metadata as a ZIP for official records

Date range selector (applies to all charts on the page)
Zone multi-select filter
Camera multi-select filter (dependent on zone)
Comparison toggle: compare current period vs previous period

4.4.3 Report Export
Export Analytics as PDF report with all visible charts rendered at print resolution
Export raw data as CSV for custom analysis in external tools
Scheduled reports: configure automatic weekly/monthly email delivery to specified recipients

4.5 Cameras Page
The Cameras page is the administrative view for managing all connected camera feeds. Only Admins and Operators have write access.

4.5.1 Camera Inventory Table

4.5.2 Add / Edit Camera Form
Camera Name (required)
Zone assignment (required, dropdown with option to create new zone)
Stream URL — RTSP/RTMP/HTTP-MJPEG (required, with 'Test Connection' button)
GPS Coordinates — auto-fill via map pin or manual entry
Frame sampling rate — default 5 fps, configurable 1–30 fps
Detection sensitivity — confidence threshold slider (0.3–0.9, default 0.6)
Alert routing — assign default officer(s) for alerts from this camera

4.5.3 Camera Health Panel
Per-camera uptime percentage (last 30 days)
Stream quality metrics: dropped frames %, average latency
Last 10 connection events log (connects, disconnects, errors)

4.6 Settings Page
Settings contains system-wide configuration. Only accessible to Admins, with some sections visible to Operators.

4.6.1 User Management (Admin only)
List all users: name, email, role, last active, account status
Invite new user via email (sends invite link with role pre-assigned)
Edit user role, zone assignments, and notification preferences
Deactivate / reactivate user accounts

4.6.2 Notification Settings
Global alert channels: Email, SMS, In-App — toggle each on/off
Per-zone notification routing — map zones to officers and supervisors
Alert throttling: suppress duplicate alerts from the same camera within N minutes
Quiet hours: disable SMS notifications during specified time windows
Test notification button — send a sample alert to verify configuration

4.6.3 Zone Management
Create, rename, and delete zones
Assign cameras to zones (drag-and-drop or multi-select)
Define zone boundaries on the map for geographic filtering

4.6.4 System Configuration (Admin only)
AI model version management — view active model version, rollback option
Default confidence threshold for detection (global override)
Data retention policy — configure how long incident images and logs are kept
API key management for third-party integrations


5. Global UI Components

5.1 Navigation Sidebar
Fixed left sidebar, 240px wide on desktop, collapsible to icon-only (64px) mode
Items: Overview, Live Monitoring, Incidents (with badge count for open alerts), Analytics, Cameras, Settings
Bottom: User avatar + name, role badge, and logout button
Active page highlighted; hover tooltips in collapsed mode

5.2 Global Alert Banner
Sticky banner at the top of every page that shows count of active/open alerts
Click to expand a quick-list of the 5 most recent unresolved alerts
Color: amber background for 1–5 open alerts, red background for 6+ open alerts

5.3 Notification Toast
Non-blocking toast notifications appear bottom-right for new alert events
Show: camera name, zone, object detected, time, and a Dismiss / View button
Auto-dismiss after 8 seconds; persists on hover
Maximum 3 toasts visible simultaneously; older ones stack underneath

5.4 Search
Global search (Cmd+K shortcut) opens a command palette
Search across: incidents by ID, cameras by name, zones, and user names
Recent searches persisted in localStorage

7. Non-Functional Requirements
7.1 Performance
Initial page load (LCP): < 2.5 seconds on 4G connection
Real-time alert delivery to dashboard: < 2 seconds from detection event
Dashboard remains responsive with up to 16 simultaneous live video feeds
Analytics charts render in < 1 second for queries up to 1 million incident records
Incident table pagination handles 100,000+ records without degradation
7.2 Responsiveness
Fully functional on desktop (1280px+) and tablet (768px–1279px)
Read-only mobile view (375px+) for field officers: alert feed, incident detail, and map only
Live video grid degrades gracefully: 2×2 on tablet, single feed on mobile
7.3 Accessibility
WCAG 2.1 Level AA compliance throughout
All interactive elements keyboard navigable with visible focus indicators
Screen reader support for alert notifications and status changes
Color is never the sole means of conveying status — always paired with text or icon
Minimum contrast ratio 4.5:1 for normal text, 3:1 for large text
7.4 Security
All dashboard traffic served over HTTPS (TLS 1.3 minimum)
JWT session tokens expire after 8 hours; refresh tokens valid for 30 days
RBAC enforced on all API routes — not just UI elements
Audit log records all privilege-sensitive actions (role change, camera delete, user deactivation)
CSP headers configured to prevent XSS; no inline scripts
7.5 Browser Support
Chrome 120+, Firefox 120+, Safari 16+, Edge 120+ (latest 2 major versions)
No Internet Explorer support
